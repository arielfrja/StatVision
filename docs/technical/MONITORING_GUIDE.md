# 📜 StatVision Logging & Monitoring Guide

This guide explains how to monitor the three core parts of the StatVision infrastructure using both the Google Cloud Console (GUI) and the Terminal (CLI).

---

## 1. API Service (`statvision-api-test`)
Handles user requests, video uploads, and triggers the AI pipeline.

### 🖥️ GUI (Cloud Console)
1. Go to [Cloud Run Console](https://console.cloud.google.com/run).
2. Click on **`statvision-api-test`**.
3. Click the **LOGS** tab.
4. *Tip:* Use the "Log type" filter to see only "Request logs" or "App logs".

### 💻 Terminal (gcloud CLI)
```bash
# See last 50 lines of logs
gcloud logging read "resource.type=cloud_run_revision AND resource.labels.service_name=statvision-api-test" --limit=50 --project=statsvision-477017

# Stream logs in real-time (Tail)
gcloud alpha run services logs tail statvision-api-test --project=statsvision-477017
```

## 2. Worker Service (`statvision-worker-test`)
The "Brains" of the operation. Handles FFMPEG slicing and Gemini AI analysis.

### 🖥️ GUI (Cloud Console)
1. Go to [Cloud Run Console](https://console.cloud.google.com/run).
2. Click on **`statvision-worker-test`**.
3. Click the **LOGS** tab.
4. Look for prefixes like `[ORCHESTRATOR]` (slicing) or `[CHUNK_PROCESSOR]` (AI analysis).

### 💻 Terminal (gcloud CLI)
```bash
# Filter for AI Analysis errors
gcloud logging read "resource.labels.service_name=statvision-worker-test AND textPayload:\"CHUNK_PROCESSOR\" AND severity>=ERROR" --limit=20 --project=statsvision-477017

# See the last 10 minutes of orchestration logs
gcloud logging read "resource.labels.service_name=statvision-worker-test AND textPayload:\"ORCHESTRATOR\"" --limit=50 --project=statsvision-477017
```

## 3. Cloud Tasks (The Smart Queue)
The conveyor belt that manages the flow between the API and the Workers.

### 🖥️ GUI (Cloud Console)
1. Go to [Cloud Tasks Console](https://console.cloud.google.com/cloudtasks).
2. You will see two queues:
   - `orchestrate-queue`: Handles initial video slicing.
   - `analyze-queue`: Handles per-chunk AI analysis.
3. Click on a queue name to see **Pending Tasks** or **Failing Tasks**.
4. Click the **LOGS** link on the far right of the queue row to see delivery failures.

### 💻 Terminal (gcloud CLI)
```bash
# List all pending tasks in the analysis queue
gcloud tasks list --queue=analyze-queue --location=us-central1 --project=statsvision-477017

# Check why a queue might be paused or hitting limits
gcloud tasks queues describe analyze-queue --location=us-central1 --project=statsvision-477017
```

## 🚀 Pro-Tips for Debugging

| If you see... | Check this Log Part |
| :--- | :--- |
| **"Network Error"** after upload | **API Logs** (Check for Cloud Tasks permission errors). |
| **Progress bar stuck** at 0% | **Worker Logs** (Check if FFMPEG is crashing or GCS download failed). |
| **Progress bar stuck** at ~20% | **Cloud Tasks Console** (Check if `analyze-queue` is hitting Gemini rate limits). |
| **"AI Could not start"** | **Worker Logs** (Check `[ORCHESTRATOR]` logs for slicing errors). |
